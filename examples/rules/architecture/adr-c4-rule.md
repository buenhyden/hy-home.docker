# Architecture — ADR/C4 Rule

## Case
- **[REQ-ADR-01]** Record significant architectural decisions as ADRs.
- **[REQ-ADR-02]** Keep decision rationale and consequences explicit.
- **[REQ-ARC-04]** Use C4-style diagrams for architecture communication.

## Style
- **[PROC-ADR-01]** Update ADR status lifecycle when decisions evolve.
- **[REQ-ADR-08]** Keep architecture documentation synchronized with reality.
- **[BAN-ADR-01]** Do not apply high-impact architectural changes without ADR traceability.

## Validation
- [ ] High-impact architecture decisions have ADR coverage.
- [ ] C4 diagrams map correctly to implemented boundaries.
- [ ] ADR status reflects current accepted/replaced decisions.
