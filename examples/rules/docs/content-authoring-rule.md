# Docs — Content Authoring Rule

## Case
- **[REQ-DOC-CON-01]** Keep purpose, audience, and expected outcomes explicit.
- **[REQ-DOC-TECH-01]** Preserve technical accuracy and implementation realism.
- **[REQ-DOC-8SK-01]** Use a consistent sectioned structure for maintainability.

## Style
- **[REQ-DOC-MUST-01]** Prefer active voice and concrete language.
- **[REQ-DOC-MUST-02]** Include essential setup/use sections when documenting workflows.
- **[BAN-DOC-FLF-01]** Avoid filler and marketing-style wording in technical docs.

## Validation
- [ ] Content is scoped and audience-appropriate.
- [ ] Technical claims are verifiable against workspace reality.
- [ ] Structure remains consistent and scannable.
