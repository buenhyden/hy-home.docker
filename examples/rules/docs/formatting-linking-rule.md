# Docs — Formatting & Linking Rule

## Case
- **[REQ-DOC-FMT-01]** Apply consistent heading, list, and code-block formatting.
- **[REQ-DOC-IDX-02]** Keep links stable and discoverable.
- **[REQ-DOC-VER-01]** Validate references before completion.

## Style
- **[PROC-DOC-BREAK]** Fix broken internal links immediately in editable scope.
- **[BAN-DOC-LAT-01]** No ambiguous "latest" references without version/date context.
- **[BAN-FMT-HTM-01]** Avoid malformed formatting that harms readability.

## Validation
- [ ] No broken relative links in changed docs.
- [ ] Formatting conventions are consistent.
- [ ] References are precise and non-ambiguous.
