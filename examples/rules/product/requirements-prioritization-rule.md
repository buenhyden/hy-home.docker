# Product — Requirements Prioritization Rule

## Case
- **[REQ-WF-01]** Clarify requirements before implementation begins.
- **[REQ-WF-02]** Define explicit non-goals and scope boundaries.
- **[REQ-COM-01]** Prioritize work by impact, risk, and feasibility signals.

## Style
- **[PROC-WF-01]** Keep requirement artifacts concise, testable, and traceable.
- **[REQ-COM-03]** Preserve decision rationale for priority changes.
- **[BAN-COM-02]** Avoid implicit scope expansion without explicit decision record.

## Validation
- [ ] Requirements are actionable and unambiguous.
- [ ] Prioritization rationale is documented.
- [ ] Non-goals are explicit for scope containment.
