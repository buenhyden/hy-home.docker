# Gateway Tier (01-gateway) Technical Specification

## Overview

This specification defines the technical design and implementation details for the `01-gateway` tier. It documents the concrete settings, middleware chains, security controls, and verification methods required for the Traefik and Nginx hybrid configuration.

## Strategic Boundaries & Non-goals

- **Owns**:
  - Traefik static/dynamic configuration files (`traefik.yml`, `dynamic/*.yml`).
  - Nginx proxy configuration (`nginx.conf`).
  - TLS certificate binding and protocol optimization.
- **Does Not Own**:
  - Automatic certificate renewal logic (owned by external scripts or Certbot).
  - Backend service healthcheck implementation (owned by individual applications).

## Contracts

- **Config Contract**:
  - Traefik Entrypoints: `web` (80), `websecure` (443), `metrics` (8082).
  - Traefik Providers: `docker`, `file` (/dynamic).
- **Security Contract**:
  - TLS: v1.2 min, v1.3 preferred.
  - ForwardAuth: `oauth2-proxy:4180/oauth2/auth`.
  - BasicAuth: Based on Docker secrets (`traefik_basicauth_password`).

## Core Design

### Component Boundary

- **Traefik (v3.7.6)**: root-active edge routing, TLS termination, and SSO middleware application.
- **Nginx (Alpine)**: profile-only leaf for header manipulation and proxy pass behavior on special paths such as `/keycloak/` and `/minio/`.

### Key Dependencies

- **Docker Socket**: Service discovery through a `ro` mount.
- **Shared Secrets**: Certificate and password files generated by `scripts/operations/gen-secrets.sh`.

### Tech Stack

- Traefik v3.7.6
- Nginx Alpine
- TLS 1.2/1.3

## Data Modeling & Storage Strategy

- The gateway runs as a fully stateless component.
- Configuration is managed in Git and mounted as volumes during deployment.

## Interfaces & Data Structures

### Traefik Middleware Contract (dashboard-auth)

```yaml
dashboard-auth:
  basicAuth:
    usersFile: "/run/secrets/traefik_basicauth_password"
```

## API Contract

Gateway statistics and monitoring follow the Prometheus format, and a separate API Spec is not currently required. The Traefik Dashboard is exposed only for internal administration.

## Verification

Required static verification steps for Gateway tier.

```bash
# Root-included Traefik compose context
HYHOME_COMPOSE_PROFILES=core bash scripts/validation/validate-docker-compose.sh

# Gateway hardening contract
bash scripts/hardening/check-all-hardening.sh 01-gateway

# Runtime-only checks after an approved stack is running
docker compose exec traefik traefik healthcheck --ping
docker compose exec nginx nginx -t
```

## Success Criteria & Verification Plan

- **VAL-SPC-001**: Every HTTP request is redirected to HTTPS with status `301`.
- **VAL-SPC-002**: The Traefik Dashboard is accessible only through Basic Auth.
- **VAL-SPC-003**: Paths protected by OAuth2 Proxy middleware redirect unauthenticated requests to the login page.
- **VAL-SPC-004**: Service-local standalone compose rendering is not used as gateway readiness evidence; use the root validator or an explicit runtime context instead.
