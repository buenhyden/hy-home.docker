# MongoDB Operations Policy

## Overview

이 정책은 `hy-home.docker`의 선택 NoSQL 서비스인 MongoDB replica set 운영 기준을 정의한다. 기준은 현재 tracked compose의 `mongo:8.2.11-noble`, `mongo-express:1-18-alpine3.19`, `percona/mongodb_exporter:2.37`, `MyReplicaSet`, `mongo-key` named volume, Docker Secret 기반 root/admin credential 구성이다.

## Policy Scope

- `infra/04-data/nosql/mongodb/docker-compose.yml`
- `mongo-key-generator`, `mongodb-rep1`, `mongodb-rep2`, `mongodb-arbiter`, `mongo-init`, `mongo-express`, `mongodb-exporter`
- `mongo-key`, `mongodb1-data`, `mongodb2-data`, `mongodb3-data`
- `mongodb_root_password`, `mongo_express_basicauth_password`, `MONGODB_ROOT_USERNAME`, `MONGO_EXPRESS_CONFIG_BASICAUTH_USERNAME`
- Traefik route `mongo-express.${DEFAULT_URL}` and exporter port `${MONGO_EXPORTER_PORT:-9216}`
- Linked guide and runbook under `docs/05.operations`

## Controls

- **Required**: Documentation must describe the current replica set as two data-bearing nodes plus one arbiter, initialized by `mongo-init`.
- **Required**: Replica set name references must use the compose-declared `MyReplicaSet`; no undocumented replica-set-name control is allowed.
- **Required**: Root and Mongo Express password guidance must reference Docker Secret mounts, not plaintext values.
- **Required**: Keyfile guidance must reference the `mongo-key` named volume generated by `mongo-key-generator`.
- **Allowed**: Read-only `rs.status()`, `rs.conf()`, logs, compose config, and exporter readiness checks for evidence capture.
- **Allowed**: Documentation-only corrections that keep image tags, service names, profiles, and links aligned with compose.
- **Disallowed**: Destructive resync, data directory deletion, forced election, keyfile rotation, or backup restore steps without separate approval and verified runbook evidence.
- **Disallowed**: Treating `mongodb-arbiter` as a backup/data node or exposing MongoDB directly through host ports not declared by compose.

## Exceptions

N/A - no currently approved exceptions.
