### RAG Workflow Usage (08-ai)

> Retrieval-Augmented Generation with Open WebUI & Qdrant (08-ai)

#### Overview

이 가이드는 로컬 문서를 벡터화하여 저장하고, LLM이 관련 정보를 참고하여 답변하도록 하는 RAG 시스템의 구성 방법을 설명한다.

#### Core Components

- **Source**: PDF, Markdown, 텍스트 문서 등.
- **Embedding Model**: 텍스트를 벡터로 변환 (default: `qwen3-embedding`).
- **Vector DB**: 변환된 벡터와 메타데이터를 저장 및 검색 (`Qdrant`).
- **Orchestrator**: 사용자의 질문에 맞춰 관련 문서를 검색하고 프롬프트에 주입 (`Open WebUI`).

#### Implementation Flow

##### 1. 벡터 데이터베이스 연동

- Open WebUI 설정에서 `VECTOR_DB_URL`이 `http://qdrant:6333`을 가리키도록 설정되어 있는지 확인한다.

##### 2. 문서 임베딩 및 업로드

- Open WebUI 인터페이스를 통해 문서를 업로드하면 내부적으로 Ollama의 임베딩 엔진을 호출한다.
- 임베딩된 데이터는 Qdrant의 지정된 컬렉션에 저장된다.

##### 3. 검색 및 응답 (Retrieval)

- 사용자 질문 발생 시 Open WebUI는 Qdrant에서 유사도가 높은 Top-K 문서를 검색한다.
- 검색된 컨텍스트를 시스템 프롬프트에 포함하여 LLM에 전달한다.

#### Optimization Strategies

- **Chunking**: 문서 분할 크기를 조정하여 검색 정확도와 컨텍스트 길이를 최적화한다.
- **Hybrid Search**: 단순 벡터 검색 외에 키워드 기반 검색을 병행하여 정확도를 높인다.

#### Troubleshooting

- **No Context Retrieved**: Qdrant 컬렉션이 비어 있거나 필터 조건이 너무 엄격하지 않은지 확인한다.
- **Embedding Timeout**: 대량의 문서 업로드 시 Ollama 임베딩 엔진의 처리 지연 여부를 확인한다.
