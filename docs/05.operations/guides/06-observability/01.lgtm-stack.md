# 01.Lgtm Stack Operations

## Usage

### LGTM Stack Usage

> Monitoring & Observability Architecture (Loki, Grafana, Tempo, Prometheus).

#### Overview (KR)

이 가이드는 `hy-home.docker`의 핵심 모니터링 체계인 LGTM 스택의 아키텍처와 운영 방법을 설명한다. 지표(Metrics), 로그(Logs), 트레이스(Traces)를 통합하여 시스템 전반의 가시성을 확보하는 것을 목표로 한다.

#### Strategic Boundaries

- **Loki**: 로그 집계 시스템. 레이블 기반 인덱싱을 통해 효율적인 검색 제공. (MinIO S3 백엔드)
- **Grafana**: 통합 시각화 및 알람 대시보드. Keycloak SSO 연동.
- **Tempo**: 분산 트레이싱 백엔드. 마이크로서비스 간 요청 흐름 추적. (MinIO S3 백엔드)
- **Prometheus**: 시계열 메트릭 저장소 및 알람 엔진.

#### Core Workflows

##### 1. Unified Visualization

모든 데이터는 Grafana (`https://grafana.${DEFAULT_URL}`)를 통해 조회한다. Explore 메뉴에서 LogQL, PromQL, TraceQL을 사용하여 상관 분석이 가능하다.

##### 2. Storage Integration (MinIO)

Loki와 Tempo는 대용량 데이터 처리를 위해 S3 호환 저장소인 MinIO를 사용한다.

- **Buckets**: `loki-data`, `tempo-data`
- **Auth**: Docker Secrets (`minio_app_user_password`) 사용

#### Operational Tasks

##### Dashboard Provisioning

`infra/06-observability/grafana/dashboards` 경로에 JSON 파일을 추가하면 Grafana 기동 시 자동으로 로드된다.

##### Alerting Rule Management

Prometheus 알람 규칙은 `infra/06-observability/prometheus/config/alert_rules`에서 YAML 파일로 관리한다.

#### Troubleshooting

- **No Data in Dashboards**: Alloy 콜렉터 상태(`https://alloy.${DEFAULT_URL}`)를 먼저 확인한다.
- **SSO Login Failure**: Keycloak 서비스 상태 및 OAuth2 클라이언트 시크릿 일치 여부를 확인한다.
