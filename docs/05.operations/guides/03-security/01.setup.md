# 01.Setup Operations

> Migrated from `docs/05.operations/03-security/01.setup.md` during the 2026-05-10 operations taxonomy consolidation.

## Usage

### Security Setup Usage (03-security)

> Vault Secret Management Setup & Initialization (03-security)

#### Overview

이 가이드는 `hy-home.docker` 플랫폼의 보안 루트(03-security)인 HashiCorp Vault를 설정하고 초기화하는 과정을 설명한다. 데이터 암호화, 시크릿 관리, 그리고 초기 Unseal 절차를 다룬다.

#### Prerequisites

- `infra/01-gateway` (Traefik)가 정상 작동 중이어야 함 (Vault UI 접속용).
- `${DEFAULT_SECURITY_DIR}/vault/data` 디렉터리가 생성되어 있어야 함 (Raft 저장소).
- `docker compose` 명령을 실행할 권한이 있어야 함.

#### Step-by-Step Instructions

##### 1. Vault 서비스 실행

```bash
cd infra/03-security/vault
docker compose up -d
```

- 서비스 실행 후 `https://vault.${DEFAULT_URL}` 접속 확인.
- 초기 상태는 `Initialized: false`, `Sealed: true`임.

##### 2. Vault 초기화 (Initialization)

Vault를 처음 실행할 때 한 번 진행한다.

```bash
docker exec -it vault vault operator init
```

> [!IMPORTANT]
> **출력되는 `Unseal Key` 5개와 `Initial Root Token`을 안전한 곳에 별도로 저장한다.**
> 이 정보를 분실하면 Vault 내부의 모든 데이터를 영구적으로 읽을 수 없다.

##### 3. Vault Unseal 절차

Vault는 재시작할 때마다 '봉인(Sealed)' 상태가 된다. 이를 해제하기 위해 키 조각 중 3개가 필요하다.

```bash
docker exec -it vault vault operator unseal <KEY_1>
docker exec -it vault vault operator unseal <KEY_2>
docker exec -it vault vault operator unseal <KEY_3>
```

- `Sealed: false`가 되면 서비스가 시크릿을 읽을 수 있는 상태가 된다.

##### 4. Vault Agent 연동 (선택 사항)

측면 앱(Sidecar)에서 시크릿을 자동으로 주입받으려면 `vault-agent` 설정을 수행한다.

1. `vault/config/vault-agent.hcl` 설정 확인.
2. `bash scripts/bootstrap-vault-approle.sh`를 실행하여 템플릿 읽기 권한(AppRole) 자동 할당.

#### Validation

- [ ] `vault status` 명령어로 `Sealed: false` 확인.
- [ ] UI(`https://vault.${DEFAULT_URL}`)에 Root Token으로 로그인 가능 여부.
- [ ] `vault kv put secret/test value=hello`로 테스트 시크릿 생성 및 조회 확인.

#### Troubleshooting

- **Cluster Unreachable**: Raft 통신 포트(8201)가 `infra_net` 내에서 열려 있는지 확인한다.
- **Permission Denied**: `IPC_LOCK` capability가 Docker에서 활성화되어 있는지 `docker-compose.yml`을 확인한다.
- **Snapshot Failure**:`/vault/data` 권한이 Vault 실행 사용자(`vault`)에게 있는지 확인한다.
