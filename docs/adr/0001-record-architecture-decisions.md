# 0001 - Record Architecture Decisions

## Status

Accepted

## Context

이 프로젝트는 다양한 인프라 구성 요소와 운영 선택지가 존재합니다. 변경 이유와 트레이드오프를 기록하지 않으면, 향후 유지보수 시 의사결정의 맥락을 잃기 쉽습니다.

## Decision

모든 주요 아키텍처 결정은 `docs/adr/`에 ADR 형식으로 기록합니다.

## Consequences

- 결정 배경과 이유를 추적할 수 있습니다.
- 신규 참여자가 맥락을 빠르게 이해할 수 있습니다.
- 문서 유지 보수 비용이 소폭 증가합니다.
