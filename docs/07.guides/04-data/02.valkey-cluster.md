# Valkey Cluster Guide (04-data)

> Distributed Valkey Cluster Setup & Maintenance (04-data)

## Overview

이 가이드는 6개의 노드로 구성된 분산 Valkey 클러스터의 초기화 및 운영 방법을 설명한다. 데이터 샤딩(Sharding)과 복제(Replication)를 통한 성능 확장을 다룬다.

## Topology

- **Primaries**: 3개 노드 (Slots 0-16383 분산)
- **Replicas**: 3개 노드 (각 Master당 1개의 복제본)
- **Networking**: 각 노드는 서비스 포트(6379+)와 클러스터 버스 포트(16379+)를 사용함.

## Maintenance Procedures

### 1. 클러스터 초기화

```bash
cd infra/04-data/valkey-cluster
docker compose up -d
```

- `valkey-cluster-init` 작업 컨테이너가 실행되어 노드들을 클러스터로 묶고 슬롯을 할당한다.

### 2. 노드 상태 모니터링

```bash
# 모든 노드와 슬롯 할당 상태 확인
docker exec valkey-node-0 valkey-cli -p 6379 cluster nodes

# 클러스터 슬롯 정보 요약
docker exec valkey-node-0 valkey-cli -p 6379 cluster info
```

### 3. 노드 장애 및 복구

- 마스터 노드 장애 시 레플리카가 자동으로 승격(Promotion)된다.
- 장애 노드 복구 후 클러스터에 재참여시키면 자동으로 레플리카로 설정된다.

## Best Practices

- **Memory Policy**: `allkeys-lru` 또는 `volatile-lru` 설정을 통해 메모리 부족 시 오래된 데이터를 삭제하도록 구성한다.
- **Client Library**: 클러스터를 지원하는 클라이언트 모드(Cluster Mode)를 반드시 활성화해야 한다.

## Troubleshooting

- **Cluster Down**: 전체 슬롯(16384개) 중 할당되지 않은 슬롯이 있는 경우 클러스터가 중단된다. 재균형(Rebalance)이 필요하다.
- **Slot Redirection**: `(error) MOVED` 응답이 오면 클라이언트가 해당 키가 위치한 노드로 리디렉션해야 함을 의미한다.
