# High Availability PostgreSQL

> Advanced guide for managing the Patroni-managed PG cluster.

## Overview

The `postgresql-cluster` utilizes Patroni for lifecycle management, ETCD for consensus, and HAProxy for request routing. This ensures zero-downtime failover for data-intensive applications.

## Cluster Management

### Monitoring Health

Use the `patronictl` tool to inspect the cluster state:

```bash
docker compose exec pg-0 patronictl list
```

### Manual Failover (Switchover)

When planned maintenance is required:

```bash
docker compose exec pg-0 patronictl switchover
```
