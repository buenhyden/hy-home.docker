# Core Infrastructure Databases

> Comprehensive guide for managing the shared SQL and Cache layer.

## Overview

The `mng-db` stack provides essential persistence for control-plane services like Keycloak and n8n. This guide covers common management tasks for these shared instances.

## Services Covered

1. **PostgreSQL (mng-pg)**: Version 17, used for most infrastructure tool metadata.
2. **Valkey (mng-valkey)**: Version 9.0, used for shared sessions and caching.
3. **Valkey Cluster**: High-availability distributed cache for application workloads.

## Getting Started

### Accessing Shared Postgres

To connect for manual maintenance:

```bash
docker exec -it mng-pg psql -U admin -d template1
```

### Accessing Valkey

Use RedisInsight at `https://redisinsight.${DEFAULT_URL}` for a visual interface, or CLI:

```bash
docker exec -it mng-valkey valkey-cli
```
