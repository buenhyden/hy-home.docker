# Object & Distributed Storage

> Guide for MinIO and SeaweedFS storage layers.

## 1. Context & Objective

Object storage provides S3-compatible endpoints for large-scale blob storage, logs, media, and distributed file systems.

- **Infrastructure Storage**: MinIO (Used by Loki, Tempo)
- **Distributed System**: SeaweedFS (Optimized for small files and high throughput)

## 2. Infrastructure Overview

### 2.1 MinIO (S3)

High-performance object storage with automated bucket management.

- **Buckets**: `tempo`, `loki`, `cdn` (Initialized by `minio-init`)
- **UI**: `https://minio-admin.${DEFAULT_URL}`

### 2.2 SeaweedFS

Distributed storage system with master/volume/filer architecture.

- **Master**: Metadata and cluster coordination.
- **Volume**: Persistence layer for actual data.
- **Filer**: S3-gateways and distributed file system interface.

## 3. Operations & Maintenance

### MinIO CLI (mc)

Interact with MinIO using the official CLI:

```bash
read -r MINIO_ACCESS_KEY
read -rsp "MinIO secret key: " MINIO_SECRET_KEY; echo
docker run --rm -it minio/mc alias set local http://minio:9000 "$MINIO_ACCESS_KEY" "$MINIO_SECRET_KEY"
unset MINIO_SECRET_KEY
```

민감값을 명령어에 직접 적거나 문서에 남기지 않는다. 가능한 경우 승인된 secret 주입 절차를 사용한다.

### SeaweedFS Health

Check volume server status:

```bash
curl -s http://seaweed-master:9333/cluster/status
```

## 4. Best Practices

- **Bucket Policies**: Use restrictive policies for internal storage and only expose `cdn` or public assets.
- **File Consolidation**: SeaweedFS excels at managing billions of small files efficiently.
- **Loki/Tempo**: Ensure `minio` is healthy before starting observability stacks.

## 5. Troubleshooting

- **Slow I/O**: Check volume server disk and network throughput in SeaweedFS.
- **Init Failures**: Ensure `minio-init` has completed if buckets are missing.
- **Certificate Errors**: Both systems use TLS; verify certificate paths in `docker-compose.yml`.
