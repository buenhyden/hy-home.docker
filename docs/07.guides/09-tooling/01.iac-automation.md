# IaC Automation Guide (09-tooling)

> Managed Terraform Workflows via Terrakube (09-tooling)

## Overview

이 가이드는 Terrakube를 사용하여 인프라를 코드로 관리(IaC)하고, 협업 및 자동화된 배포 프로세스를 구성하는 방법을 설명한다.

## Architecture

- **Terrakube UI**: 워크스페이스 및 실행 이력 관리 인터페이스.
- **Terrakube API**: 작업을 스케줄링하고 상태를 관리하는 백엔드.
- **Terrakube Executor**: Docker 기반의 격리된 환경에서 Terraform 명령(plan, apply)을 실행.
- **Remote State**: MinIO (S3)를 백엔드로 사용하여 `terraform.tfstate` 파일을 안전하게 보관.

## Step-by-Step Instructions

### 1. 워크스페이스 구성

- **Source Control**: Git 저장소를 연결하여 Terraform 코드를 동기화한다.
- **Variables**: AWS, Azure 또는 로컬 인프라 접근을 위한 환경 변수를 Terrakube 내부 비밀로 등록한다.

### 2. 실행 및 승인 프로세스

- **Plan**: 코드 변경 발생 시 자동으로 `terraform plan`이 실행된다.
- **Review**: UI에서 변경 사항을 검토한 후 수동으로 승인(Approve)을 수행한다.
- **Apply**: 승인된 작업에 한해 실제 인프라 반영이 시작된다.

## Maintenance

- **State Lock**: 작업 중 충돌 방지를 위해 Valkey를 기반으로 상태 잠금이 관리된다.
- **Module Registry**: 재사용 가능한 모듈을 Terrakube 내부 레지스트리에 등록하여 관리 효율을 높인다.

## Troubleshooting

- **Executor Hang**: 실행기가 'Running' 상태에서 멈춘 경우 Docker 소켓 연결 상태를 점검한다.
- **State Synchronization Error**: MinIO 버킷(`tfstate`) 접근 권한이나 네트워크 상태를 확인한다.
