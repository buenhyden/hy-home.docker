# Performance Testing Guide (09-tooling)

> Distributed Load Testing with Locust & InfluxDB (09-tooling)

## Overview

이 가이드는 Locust를 사용하여 플랫폼의 엔드포인트를 벤치마킹하고, 성능 병목 지점을 식별하는 방법을 설명한다.

## Test Workflow

1. **Scripting**: Python으로 `locustfile.py`를 작성하여 사용자 시나리오를 정의한다.
2. **Orchestration**: `locust-master`를 시작하고 부하량에 따라 `locust-worker`를 확장한다.
3. **Execution**: UI에서 초당 가상 사용자 수(User rate)를 설정하여 부하를 가한다.
4. **Analysis**: 실시간 지표를 시각화하고 InfluxDB에 저장된 이력을 분석한다.

## Procedures

### 1. 워커 확장 (Scaling)

부동 제어를 위해 다수의 워커가 필요한 경우:
```bash
docker compose -f infra/09-tooling/locust/docker-compose.yml up --scale locust-worker=5 -d
```

### 2. 결과 저장 및 대시보드

- Locust의 실시간 지표는 InfluxDB로 전송된다.
- Grafana에서 InfluxDB 데이터 소스를 연결하여 맞춤형 성능 대시보드를 구성할 수 있다.

## Best Practices

- **Zero-Downtime**: 운영 환경 테스트 시에는 트래픽 유입량을 서서히 늘리는 램프업(Ramp-up) 시간을 반드시 설정한다.
- **Isolation**: 부하 테스트용 전용 네트워크 환경을 구축하여 운영 트래픽과의 간섭을 최소화한다.

## Troubleshooting

- **Worker Not Connecting**: 마스터 호스트 명칭(`locust-master`) 및 네트워크 가시성을 확인한다.
- **High Resource Usage on Master**: 마스터 노드는 데이터 수집 역할만 하므로, 실제 부하 발생은 전적으로 워커에서 담당하도록 설계한다.
