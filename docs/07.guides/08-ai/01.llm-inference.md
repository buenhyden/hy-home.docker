# LLM Inference Management Guide (08-ai)

> Model Lifecycle, GPU Optimization & Inference Patterns (08-ai)

## Overview

이 가이드는 Ollama를 사용하여 로컬 LLM을 효율적으로 배포하고 관리하는 방법을 설명한다.

## Architecture

- **Ollama Engine**: GGUF 및 PyTorch 모델을 로드하여 고속 추론 수행.
- **NVIDIA CUDA**: 하드웨어 가속을 통해 GPU VRAM을 활용한 추론 가속.
- **Context Management**: 다중 사용자 요청에 대한 컨텍스트 윈도우 할당 및 토큰 스케줄링.

## Step-by-Step Instructions

### 1. 모델 관리 (Model Lifecycle)

- **Pulling Models**: 필요한 모델은 CLI를 통해 명시적으로 다운로드한다.

  ```bash
  docker exec -it ollama ollama pull llama3.1:8b
  ```

- **Custom Modelfiles**: 특정 시스템 프롬프트나 파라미터가 필요한 경우 `Modelfile`을 작성하여 맞춤형 모델을 생성한다.

### 2. 성능 최적화 (Optimization)

- **VRAM 할당**: 모델 크기에 따라 GPU 메모리가 가득 차지 않도록 레이어 수를 조정하거나 양자화(Quantization)된 모델(Q4_K_M 등)을 사용한다.
- **Concurrent Requests**: 동시 요청 수가 많을 경우 `OLLAMA_NUM_PARALLEL` 환경 변수를 조정하여 처리량을 높인다.

## Monitoring

- **Metrics**: `ollama-exporter`를 통해 다음 지표를 모니터링한다.
  - `tokens_per_second`: 추론 속도.
  - `vram_usage`: GPU 메모리 점유율.
  - `active_requests`: 현재 처리 중인 요청 수.

## Troubleshooting

- **Inference Slowdown**: GPU 가속이 비활성화되었는지 (`nvidia-smi` 확인) 또는 CPU로 오프로드되었는지 확인한다.
- **Model Loading Failed**: VRAM 부족으로 인해 모델 로딩이 실패한 경우 더 높은 양자화 비율을 가진 모델을 시도한다.
