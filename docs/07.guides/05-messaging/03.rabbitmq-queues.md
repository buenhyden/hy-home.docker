# RabbitMQ | AMQP Queues (03.rabbitmq-queues)

> Reliable task queuing and message distribution patterns.

## Overview

RabbitMQ provides a robust AMQP message broker for lightweight task queues, event distribution, and deferred processing. It is ideal for microservices communication and background job orchestration.

## Service Components

- **RabbitMQ Management**: The AMQP broker with an integrated management UI.
- **Management UI**: Visual monitoring and configuration interface (`15672`).
- **AMQP Protocol**: Standards-based messaging on port `5672`.

## Exchange Patterns

RabbitMQ supports several exchange types for different routing needs:

- **Direct**: Routing based on exact routing key matches.
- **Topic**: Pattern-based routing for complex subscription models.
- **Fanout**: Broadcasting messages to all bound queues.

## Management Tasks

### Accessing the Management UI

The UI is available at `https://rabbitmq.${DEFAULT_URL}` with the credentials stored in your Docker secrets. Use it to:

- Monitor queue depth and consumer activity.
- Declare exchanges, queues, and bindings.
- Inspect and purge message contents.

### Persistence Governance

Data is stored under `${DEFAULT_MESSAGE_BROKER_DIR}/rabbitmq`.

> [!IMPORTANT]
> RabbitMQ uses Docker Secrets for `rabbitmq_user` and `rabbitmq_password`. If these are changed, the internal Mnesia database might not reconcile existing permissions automatically.
