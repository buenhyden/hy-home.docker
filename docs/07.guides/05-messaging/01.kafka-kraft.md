# Kafka KRaft Guide (05-messaging)

> Zookeeper-less Kafka Cluster Setup & Management for hy-home.docker.

## Overview (KR)

이 가이드는 Zookeeper 없이 작동하는 KRaft(Kafka Raft) 모드 기반의 Apache Kafka 클러스터 시스템(05-messaging)의 설정 및 운영 방법을 설명한다. 대용량 이벤트 스트리밍, 메시지 스키마 거버넌스, 그리고 Kafka Connect 연동을 다룬다.

## Architecture Highlights

- **KRaft Quorum**: 3개의 브로커 노드가 쿼럼을 형성하여 메타데이터를 직접 관리.
- **Schema Governance**: Confluent Schema Registry를 통한 Avro/JSON 스키마 버전 관리 강제.
- **Observability**: Prometheus Exporter 및 Kafbat UI를 통한 실시간 모니터링.

## Step-by-Step Instructions

### 1. Cluster Execution
```bash
# Infrastructure root에서 실행
cd infra/05-messaging/kafka
docker compose up -d
```
- `kafka-init` 서비스가 가동되어 `infra-events`, `application-logs` 등의 시스템 토픽을 자동 생성한다.

### 2. Topic Management
- **CLI**: `docker exec kafka-1 kafka-topics --bootstrap-server localhost:19092 --create --topic <topic-name> --partitions 3 --replication-factor 3`
- **UI**: `https://kafbat-ui.${DEFAULT_URL}` 접속 후 GUI를 통해 생성/수정 가능.

### 3. Schema Registry Integration
- 모든 생산자(Producer)는 메시지 전송 전 Schema Registry에 스키마를 등록해야 한다.
- **Compatibility**: 기본 `BACKWARD` 정책을 준수하여 소비자의 호환성을 보장한다.

## Maintenance & Monitoring

- **Lag Monitoring**: Grafana 대시보드에서 `Consumer Lag` 지표를 상시 확인한다.
- **Log Retention**: 서비스 가용량에 따라 `retention.ms` (기본 7일) 정책을 조정한다.

## Troubleshooting

- **Quorum Failure**: 컨트롤러 노드 2개 이상 다운 시 클러스터 읽기/쓰기 중단. `09.runbooks`를 참조하여 복구한다.
- **Schema Discord**: 생산자와 레지스트리의 스키마 불일치 시 `409 Conflict` 발생. 버전을 확인한다.
