# Kafka | KRaft Cluster (01.kafka-kraft)

> High-throughput, distributed event streaming powered by KRaft.

## Overview

Kafka serves as the central event bus for `hy-home.docker`. The cluster operates in **KRaft mode**, eliminating the need for Zookeeper. It includes Schema Registry for data governance and Kafka Connect for data integration.

## Architecture

### KRaft Cluster

- **Controller Quorum**: 3-node configuration for high availability.
- **Roles**: All brokers acting as both controllers and brokers.
- **Port Mapping**:
  - `9092`: External Plaingtext (Host access).
  - `19092`: Internal Plaingtext (Service-to-service).
  - `29092`: Controller Quorum.

### Supporting Ecosystem

- **Schema Registry**: Centralized Avro/JSON Schema management (`8081`).
- **REST Proxy**: HTTP access to Kafka clusters (`8082`).
- **Kafbat UI**: Visual management and monitoring (`8080`).

## Management Tasks

### Viewing Topics & Schemas

Access the **Kafbat UI** at `https://kafka-ui.${DEFAULT_URL}`. It provides a visual interface for:

- Creating and deleting topics.
- Browsing message content (including Avro decoding).
- Managing schemas in the Schema Registry.
- Monitoring consumer group offsets.

### Persistence Governance

Each broker persistent data is stored in `${DEFAULT_MESSAGE_BROKER_DIR}/kafka/broker-*`.

> [!WARNING]
> Do not manually move or delete files in the `data/` directory. KRaft metadata and log segments are tightly coupled to broker IDs.
