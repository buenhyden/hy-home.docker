# RabbitMQ Guide (05-messaging)

> Flexible AMQP Message Brokering and Task Queuing Patterns.

## Overview (KR)

이 가이드는 `hy-home.docker` 플랫폼의 비동기 작업 큐 및 마이크로서비스 간 메시지 릴레이를 담당하는 RabbitMQ(05-messaging)의 활용 방법을 설명한다. 고가용성(HA) 큐 구성 및 AMQP 라우팅 패턴을 다룬다.

## Service Components

- **Broker Engine**: RabbitMQ 3.13 Management-enabled 컨테이너.
- **AMQP Endpoint**: Port `5672` (Application), `15672` (Management UI).
- **Persistence Layer**: `${DEFAULT_MESSAGE_BROKER_DIR}/rabbitmq` 경로에 데이터 영속성 유지.

## Core Routing Patterns

- **Direct (Task Queues)**: 특정 작업자에게 1:1로 작업을 정적 할당.
- **Topic (Pub/Sub)**: 와일드카드(`#`, `*`)를 사용하여 복잡한 이벤트 구독 관계 형성.
- **Quorum Queues**: Raft 알고리즘 기반의 복제 큐를 사용하여 하드웨어 장애 시 데이터 유실 방지.

## Operations & Management

### 1. Management Console
- **Link**: `https://rabbitmq-ui.${DEFAULT_URL}`
- **Usage**: 실시간 큐 깊이(Depth), 소비자 상태, 익스체인지 토폴로지 시각화.

### 2. Message Persistence
- **Durable**: 서버 재시작 시에도 큐/익스체인지 정의 보존.
- **Persistent**: 메시지 바디를 디스크에 기록하여 브로커 장애 시 복구 가능.

### 3. Dead Lettering (DLX/DLQ)
- 처리 실패하거나 만료된 메시지를 격리하기 위해 `dead-letter-exchange` 설정을 필수로 구성한다.

## Troubleshooting

- **Resource Alarms**: 메모리/디스크 임계치 도달 시 메시지 수신이 중체됨(Blocked). 리소스 증설 또는 큐 퍼지(Purge)를 고려한다.
- **Consumer Starvation**: 특정 큐에 소비자가 없어 메시지가 무한히 쌓이는 경우 알람 발생.
