# ksqlDB | Streaming SQL (02.ksql-streaming)

> Real-time stream processing and SQL-based transformations.

## Overview

ksqlDB provides a SQL interface for Kafka, enabling real-time stream processing, data enrichment, and materialized view creation. It integrates with Schema Registry to support typed streams.

## Service Components

- **ksqlDB Server**: The processing engine integrated with Kafka (`8088`).
- **ksqlDB CLI**: Interactive command-line interface for querying and managing streams.
- **Datagen**: Utility for generating mock data for testing streaming pipelines.

## Usage Scenarios

### Simple Stream Processing

ksqlDB allows creating streams and tables from Kafka topics using standard SQL:

```sql
CREATE STREAM orders (id INT, product STRING, price DOUBLE)
  WITH (KAFKA_TOPIC='orders_topic', VALUE_FORMAT='AVRO');
```

### Materialized Views

Create tables that automatically update as new events arrive:

```sql
CREATE TABLE total_sales AS
  SELECT product, SUM(price) FROM orders GROUP BY product;
```

## Management Tasks

### Using the CLI

The ksqlDB CLI is available as a containerized utility. Access it via:

```bash
docker compose run --rm ksql-cli ksql http://ksqldb-server:8088
```

### Resource Requirements
ksqlDB is an **optional service** controlled by the `messaging-option` profile. It requires a healthy Kafka cluster and Schema Registry to function.
