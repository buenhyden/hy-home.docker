# Setup Guide: Security Tier (03-security)

<!-- [ID:docs:07:03-security:setup] -->
: Deployment, initialization, and bootstrapping of HashiCorp Vault.

---

## Prerequisites

- **Network**: `infra_net` must exist.
- **Storage**: `${DEFAULT_SECURITY_DIR}/vault/data` must be writable by the container.
- **Tools**: `docker` and `vault` CLI (optional for host-level access).

## Deployment Steps

1.  **Deploy Vault**:
    ```bash
    cd infra/03-security/vault
    docker compose up -d
    ```

2.  **Initialize Vault** (One-time per environment):
    ```bash
    docker exec -it vault vault operator init
    ```
    > [!IMPORTANT]
    > **Securely save the 5 unseal keys and the Initial Root Token.** These cannot be recovered if lost.

3.  **Unseal Vault**:
    Vault starts in a 'sealed' state. Use 3 of your 5 unseal keys to activate it:
    ```bash
    docker exec -it vault vault operator unseal <KEY_1>
    docker exec -it vault vault operator unseal <KEY_2>
    docker exec -it vault vault operator unseal <KEY_3>
    ```

4.  **Verify Status**:
    ```bash
    docker exec -it vault vault status
    ```
    `Sealed` should be `false`.

## Bootstrapping Secrets

Once unsealed, log in with the root token to configure engines and policies:

```bash
docker exec -it vault vault login <ROOT_TOKEN>

# Enable KV Secret Engine v2
docker exec -it vault vault secrets enable -path=secret kv-v2
```

## Vault Agent (Consumer)

The `vault-agent` service is configured to automatically authenticate using AppRole or Token and sink secrets into local files or env-vars for other services.

- **Configuration**: `infra/03-security/vault/config/vault-agent.hcl`.
- **Templates**: `infra/03-security/vault/config/templates/`.
