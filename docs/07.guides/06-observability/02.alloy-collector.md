# Alloy Collector Guide (06-observability)

> Unified Telemetry Collection and OTLP Pipeline Guide

## Overview

Grafana Alloy는 `hy-home.docker` 플랫폼의 모든 텔레메트리(지표, 로그, 트레이스)를 수집하여 적절한 백엔드로 전달하는 통합 에이전트이다.

## Collection Workflow

1. **Ingestion**: 애플리케이션은 OpenTelemetry (OTLP) gRPC/HTTP 프로토콜로 Alloy에 데이터를 전송한다.
2. **Enrichment**: Alloy는 컨테이너 ID, 노드 이름 등의 메타데이터를 자동으로 주입한다.
3. **Export**: 
   - Metrics -> Prometheus
   - Logs -> Loki
   - Traces -> Tempo

## Configuration

- **Config Path**: `infra/06-observability/alloy/config/config.alloy`
- **UI (Debug)**: `https://alloy.${DEFAULT_URL}` 에 접속하여 파이프라인 그래프와 컴포넌트 상태를 실시간으로 확인할 수 있다.

## How to add new target

1. `config.alloy` 파일에 새로운 수집 컴포넌트(예: `discovery.docker`) 추가.
2. `alloy-data` 볼륨에 저장된 상태 확인.
3. `docker compose restart alloy` 실행 (Hot reload 지원 시 리로드 명령 사용).

## Troubleshooting

- **Connection Refused**: Alloy가 OTLP 포트(4317, 4318)에서 정상 수신 중인지 확인.
- **Relabeling Issues**: Alloy UI의 'Component Details'에서 메타데이터 레이블링 결과를 확인.
