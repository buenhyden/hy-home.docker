# LGTM Stack Guide (06-observability)

> Monitoring & Observability Architecture (LGTM Stack)

## Overview

이 가이드는 `hy-home.docker`의 핵심 모니터링 체계인 LGTM 스택(Loki, Grafana, Tempo, Mimir/Prometheus)의 아키텍처와 운영 방법을 설명한다.

## LGTM Architecture

1. **Prometheus (Metrics)**: 시스템 및 애플리케이션의 시계열 지표 저장 및 알람 평가.
2. **Loki (Logs)**: 레이블링 기반의 로그 생성 및 저장. (MinIO S3 백엔드 사용)
3. **Tempo (Traces)**: 요청의 흐름을 추적하는 분산 트레이싱 저장소. (MinIO S3 백엔드 사용)
4. **Grafana (Visualization)**: 모든 데이터를 하나의 화면에서 시각화하고 SSO(Keycloak) 연동.

## Core Workflows

### 1. 로그 및 지표 조회 (Grafana)

- **URL**: `https://grafana.${DEFAULT_URL}`
- **Explore 메뉴**: LogQL(Logs), PromQL(Metrics), TraceQL(Traces)을 사용하여 직접 쿼리 가능.

### 2. 저장소 구성 (MinIO Backend)

Loki와 Tempo는 데이터를 파일 시스템이 아닌 S3 호환 저장소(`04-data` 계층의 MinIO)에 저장한다.
- **Loki Bucket**: `loki`
- **Tempo Bucket**: `tempo`

## Operational Tasks

- **대시보드 추가**: `infra/06-observability/grafana/dashboards` 하위에 JSON 파일을 추가하여 자동 배포.
- **알람 규칙 수정**: `infra/06-observability/prometheus/config/alert_rules` 하위 YAML 파일 수정.

## Troubleshooting

- **데이터 유실**: MinIO 서비스 상태 및 버킷 권한을 먼저 확인한다.
- **쿼리 타임아웃**: Grafana 환경 변수 중 데이터 프록시 타임아웃 설정을 조정한다.
