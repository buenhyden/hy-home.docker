# n8n Automation Guide (07-workflow)

> Low-code Workflow Automation and Integration Guide (07-workflow)

## Overview

이 가이드는 n8n을 사용하여 외부 API와 시스템을 통합하고 시각적인 방식으로 자동화 워크플로우를 구성하는 방법을 설명한다.

## Workflow Principles

- **Visual Logic**: 노드 기반 인터페이스를 사용하여 데이터 흐름을 설계.
- **Webhook Integration**: 외부 이벤트(GitHub, Discord 등)를 실시간으로 수신하여 처리 가능.
- **Secret Management**: n8n 내부의 `Credentials` 시스템을 사용하여 안전하게 인증 정보 관리.

## Key Procedures

### 1. 워크플로우 생성 및 실행

- **Node Selection**: 필요한 트리거(Trigger)와 액션(Action) 노드를 선택.
- **Data Transformation**: `Code` 노드(JavaScript)를 사용하여 복잡한 데이터 변환 수행 가능.
- **Activation**: 설계를 마친 워크플로우를 `Active` 상태로 전환하여 자동 실행 활성화.

### 2. 에러 핸들링

- **Error Workflow**: 워크플로우 실패 시 실행될 전용 에러 핸들링 노드 구성 권장.
- **Notification**: 실패 발생 시 Slack이나 이메일로 즉시 알림 전송.

## Security Best Practices

- n8n 관리자 계정은 반드시 비밀번호를 사용하여 보호한다.
- 외부 노출이 필요한 Webhook의 경우 `01-gateway` (Traefik) 수준에서 화이트리스트 IP 기반 접근 제어를 검토한다.

## Troubleshooting

- **Workflow Interrupted**: 실행 중 서버 리소스 부족으로 중단된 경우 로그 확인 필요.
- **API Rate Limit**: 외부 API 호출 시 속도 제한(Rate Limit) 정책을 준수하는지 확인.
