# Setup Guide: Auth Tier (02-auth)

<!-- [ID:docs:07:02-auth:setup] -->
: Deployment and bootstrapping of Keycloak and OAuth2 Proxy.

---

## Prerequisites

- **Network**: `infra_net` must exist.
- **Database**: `mng-pg` (PostgreSQL) must be healthy and accessible.
- **Secrets**: Admin passwords and database credentials must be configured via `scripts/gen-secrets.sh`.

## Deployment Steps

### 1. Database Initialization
Ensure Keycloak has its own database in PostgreSQL.
- **DB Name**: `keycloak`
- **User**: `${KEYCLOAK_DB_USER}` (from `.env`)

### 2. Startup Order
1. Start PostgreSQL (if not already running).
2. Start Valkey (if not already running).
3. Deploy Keycloak:
   ```bash
   cd infra/02-auth/keycloak
   docker compose up -d
   ```
4. Deploy OAuth2 Proxy:
   ```bash
   cd infra/02-auth/oauth2-proxy
   docker compose up -d
   ```

### 3. Keycloak Bootstrapping
On first start, an admin user is created via environment variables:
- **Username**: `${KEYCLOAK_ADMIN_USER}`
- **Password**: Read from `keycloak_admin_password` secret.

### 4. OAuth2 Proxy Integration
1. Create a "Realm" in Keycloak (e.g., `hy-home`).
2. Create an "OIDC Client" for OAuth2 Proxy.
   - **Client ID**: `oauth2-proxy`
   - **Access Type**: Confidential
3. Note the **Client Secret** and update `oauth2_proxy_client_secret` in Docker Secrets.

## Verification

- [ ] **Keycloak UI**: `https://keycloak.${DEFAULT_URL}/admin/`
- [ ] **Auth Health**: `https://auth.${DEFAULT_URL}/ping` should return `OK`.
- [ ] **SSO Flow**: Try accessing a protected service; you should be redirected to Keycloak.
