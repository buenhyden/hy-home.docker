# Auth Setup Guide (02-auth)

> Identity and Access Management (IAM) & SSO Gateway (02-auth)

## Overview

이 가이드는 `hy-home.docker` 플랫폼의 인증 및 권한 관리 시스템(02-auth)을 구축하고 설정하는 과정을 설명한다. Keycloak을 이용한 Identity Provider 구성과 OAuth2 Proxy를 이용한 전역 SSO 연동을 다룬다.

## Prerequisites

- `infra/01-gateway` (Traefik)가 정상 작동 중이어야 함.
- `infra/04-data/mng-db` (PostgreSQL, Valkey)가 실행 중이어야 함.
- `.env` 파일에 `DEFAULT_URL`, `KEYCLOAK_ADMIN_USER`, `OAUTH2_PROXY_CLIENT_ID` 등이 정의되어 있어야 함.
- `scripts/gen-secrets.sh`를 통해 필요한 비밀번호와 시크릿이 생성되어 있어야 함.

## Step-by-Step Instructions

### 1. Keycloak 초기화 및 실행

```bash
cd infra/02-auth/keycloak
docker compose up -d
```

- 실행 후 `https://keycloak.${DEFAULT_URL}` 접속 확인.
- 초기 관리자 계정으로 로그인 (ID: `KEYCLOAK_ADMIN_USER`, PW: `keycloak_admin_password` 시크릿 확인).

### 2. Realm 및 OIDC Client 생성

1. **Realm 생성**: `hy-home.realm` (또는 원하는 이름) 생성.
2. **Client 생성**:
   - Client ID: `${OAUTH2_PROXY_CLIENT_ID}` (기본값: `oauth2-proxy`)
   - Client Protocol: `openid-connect`
   - Access Type: `confidential`
   - Valid Redirect URIs: `https://auth.${DEFAULT_URL}/oauth2/callback`
3. **Credentials**: 발행된 `Secret`을 복사하여 `oauth2_proxy_client_secret` 시크릿 파일에 저장.

### 3. OAuth2 Proxy 설정 및 실행

1. `infra/02-auth/oauth2-proxy/config/oauth2-proxy.cfg` 확인 (발급자 URL, 리디렉션 URL 등).
2. 서비스 실행:

   ```bash
   cd infra/02-auth/oauth2-proxy
   docker compose up -d
   ```

3. `https://auth.${DEFAULT_URL}/ping` 접속하여 `OK` 확인.

### 4. 서비스 보호 (ForwardAuth 연동)

새로운 서비스를 SSO로 보호하려면 Traefik 라벨에 다음 미들웨어를 추가한다:

```yaml
labels:
  - "traefik.http.routers.my-service.middlewares=auth-sso@file"
```

## Validation

- [ ] Keycloak 관리자 페이지 접속 가능 여부.
- [ ] `auth.${DEFAULT_URL}/oauth2/auth` 호출 시 401(미인증) 또는 202(인증됨) 반환 여부.
- [ ] 보호된 서비스 접속 시 Keycloak 로그인 페이지로 리디렉션되는지 확인.

## Troubleshooting

- **리디렉션 루프**: `oauth2-proxy.cfg`의 `cookie_domains` 설정을 확인하고, 브라우저 쿠키를 삭제 후 재시도한다.
- **인증 실패 (401)**: Keycloak Client의 `Secret`과 OAuth2 Proxy의 `OAUTH2_PROXY_CLIENT_SECRET`이 일치하는지 확인한다.
- **데이터베이스 연결 오류**: Keycloak 로그(`docker logs keycloak`)를 확인하여 PostgreSQL 연결 상태를 점검한다.
