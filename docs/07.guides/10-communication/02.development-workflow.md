# 🧪 Development Mail Workflow (MailHog)

개발 환경에서 외부 메일 유출 없이 안전하게 테스트하는 방법을 안내합니다.

## 1. 개요
개발 중인 애플리케이션에서 실제 사용자에게 이메일이 발송되는 사고를 방지하기 위해 **MailHog** SMTP 트랩을 사용합니다. 모든 SMTP 요청은 MailHog가 캡처하며, 실제 외부망으로 전달되지 않습니다.

## 2. 사용법
애플리케이션의 SMTP 설정을 다음과 같이 변경합니다:
- **Host**: `mailhog`
- **Port**: `1025`
- **Auth**: None (Default)

## 3. 캡처된 메일 확인
`https://mailhog.${DEFAULT_URL}` 웹 UI를 통해 캡처된 모든 메일의 제목, 본문, 헤더를 실시간으로 확인할 수 있습니다.

## 4. 주의사항
MailHog는 메모리에 데이터를 저장하므로 서비스 재시작 시 모든 캡처 내역이 초기화됩니다.
