# Retention and Performance Policies

This document defines the operational standards for data retention and system performance within the 06-observability tier.

## 1. Data Retention Periods

| Data Type | Standard Retention | Extended Retention | Long-term Archive |
| :--- | :--- | :--- | :--- |
| **Metrics** | 15 Days | 90 Days (Low Res) | 1 Year (Thanos/Mimir) |
| **Logs** | 7 Days | 30 Days (Audit) | 90 Days (S3 Glacier) |
| **Traces** | 24 Hours | 7 Days (Error) | N/A |
| **Profiles** | 7 Days | 14 Days | N/A |

## 2. Resource Allocation

- **Prometheus**: 4GB RAM / 2 CPU (Standard).
- **Grafana**: 1GB RAM / 1 CPU.
- **Loki**: 2GB RAM / 1 CPU (Ingester).

## 3. High Cardinality Governance

To prevent Prometheus OOM:

- Avoid adding dynamic labels (user IDs, IP addresses) to metrics.
- Monitor `prometheus_tsdb_symbol_table_size_bytes`.

## 4. Backup Strategy

- **Grafana Dashboards**: Exported as JSON in `infra/06-observability/grafana/dashboards/`.
- **MinIO Buckets**: Regular snapshot of the `observability` bucket.
