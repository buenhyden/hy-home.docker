# AI Infrastructure Tier (08-ai) Product Requirements

## Overview (KR)

이 문서는 `08-ai` 계층의 제품 요구사항을 정의한다. 로컬 LLM 추론 엔진(`Ollama`)과 RAG 인터페이스(`Open WebUI`)를 통해 사용자에게 개인정보 보호가 강화된 자립형 AI 서비스 환경을 제공하는 것을 목표로 한다.

## Vision

외부 클라우드 의존성을 최소화하고, 내부 인프라의 GPU 자원을 효율적으로 활용하여 기밀 데이터를 안전하게 처리할 수 있는 로컬 지능형 비서 및 RAG 시스템을 구축한다.

## Problem Statement

- 클라우드 LLM 사용 시 발생하는 높은 비용과 데이터 유출 우려.
- 내부 문서와 지식 베이스를 연동한 맞춤형 AI 답변 생성 능력 부족.
- GPU 자원의 중앙화된 관리 및 효율적 배분 체계 부재.

## Personas

- **AI Engineer**: 모델 배포, 양자화 최적화, RAG 파이프라인 성능 튜닝을 담당하는 관리자.
- **Internal User**: 일상적인 업무 보조, 문서 요약, 코딩 지원을 위해 채팅 인터페이스를 활용하는 팀원.
- **System Agent**: 전용 API를 통해 자동화된 워크플로우 내에서 지능형 태스크를 수행하는 에이전트.

## Key Use Cases

- **STORY-01**: 사용자는 Open WebUI를 통해 로컬에 호스팅된 Ollama 모델과 대화하고 정답을 얻을 수 있다.
- **STORY-02**: 사용자는 문서를 업로드하여 로컬 Vector DB(Qdrant)에 저장하고, 이를 기반으로 한 RAG 검색 결과를 답변에 반영할 수 있다.
- **STORY-03**: 에이전트는 Ollama API를 호출하여 구조화된 데이터 추출이나 텍스트 분석 작업을 수행할 수 있다.

## Functional Requirements

- **REQ-PRD-FUN-01**: NVIDIA Container Toolkit을 통한 하드웨어 가속(GPU) 추론 지원.
- **REQ-PRD-FUN-02**: Ollama를 통한 다양한 오픈소스 모델(Llama3, Mistral, Qwen 등) 서빙 및 관리.
- **REQ-PRD-FUN-03**: Open WebUI를 통한 사용자 친화적인 채팅 브라우저 인터페이스 제공.
- **REQ-PRD-FUN-04**: 내부 문서 검색을 위한 RAG 파이프라인 및 Embedding 모델 지원.
- **REQ-PRD-FUN-05**: 멀티유저 지원 및 Keycloak 기반의 SSO 연동.

## Success Criteria

- **REQ-PRD-MET-01**: 로컬 모델 추론 지연시간(TTFT)이 1초 이내(Llama-3-8B 기준).
- **REQ-PRD-MET-02**: 모든 데이터(프롬프트, 문서)가 외부 네트워크로 유출되지 않고 내부망 내에서 처리됨.

## Scope and Non-goals

- **In Scope**:
  - LLM 추론 엔진(Ollama) 구성 및 GPU 연동.
  - RAG 오케스트레이션(Open WebUI) 및 UI 제공.
  - 모델 영구 저장소 및 권한 관리.
- **Out of Scope**:
  - 대규모 모델의 파인튜닝 또는 사전 학습(Training).
  - 대외 서비스용 고성능 LLM API 제공 (내부용 집중).

## Risks, Dependencies, and Assumptions

- **Risks**: GPU VRAM 부족으로 인한 OOM(Out of Memory) 발생 및 서비스 중단.
- **Dependencies**: NVIDIA GPU 드라이버 및 Container Toolkit 설치 필수. `04-data` 계층의 Qdrant(Vector DB) 가용성.

## AI Agent Requirements (If Applicable)

- **Allowed Actions**: 모델 리스트 조회, 채팅 생성, 임베딩 생성, 문서 업로드.
- **Disallowed Actions**: 호스트 시스템의 물리적 파일 수정, 외부 인터넷으로의 기밀 프롬프트 전송.
- **Evaluation Expectation**: 답변의 정확성(Faithfulness) 및 검색 결과와의 일치도(Answer Relevance).
