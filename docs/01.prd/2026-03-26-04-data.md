# PRD: Data Tier (04-data)

## Overview (KR)

`04-data` 티어는 `hy-home.docker` 플랫폼의 모든 상태 정보를 저장하고 관리하는 통합 데이터 계층이다. 트랜잭션 수반 SQL, 고속 캐싱 KV, 비정형 오브젝트 데이타, AI를 위한 벡터 데이터 등 다양한 데이터 모델을 수용하며, 고가용성(HA)과 무결성을 보장하는 것을 목표로 한다.

## Vision

애플리케이션이 데이터 저장소의 물리적 배치나 장애 여부에 구애받지 않고 일관된 인터페이스를 통해 데이터를 영속화할 수 있는 복원력 있는 데이터 허브를 제공한다.

## Target Personas

- **Backend Developers**: 안정적인 데이터 API(SQL, Redis, S3)가 필요한 서비스 개발자.
- **Data Engineers**: 데이터 파이프라인 및 분석 환경을 구축하는 기술자.
- **AI Agents**: 지식 기반(RAG) 검색을 위해 벡터 엔진이 필요한 에이전트.

## Key Use Cases

1. **Transactional Storage**: Patroni 기반 PostgreSQL 클러스터를 통한 안정적인 사용자/비즈니스 데이터 관리.
2. **Distributed Caching**: Valkey 클러스터를 이용한 대규모 세션 및 임시 데이터의 고성능 처리.
3. **S3-Compatible Storage**: MinIO를 통한 정적 자서 및 미디어 파일의 오브젝트 스토리지 제공.
4. **Vector Knowledge Base**: Qdrant를 통한 AI/RAG 목적의 벡터 임베딩 저장 및 유사도 검색.

## Functional Requirements

- **HA (High Availability)**: 주요 DB(SQL, Valkey)는 클러스터링을 통해 싱글 포인트 장애(SPOF)를 제거함.
- **Security**: 모든 DB 접근은 Docker Secrets 및 Vault 연동을 통한 인증을 필수함.
- **Isolation**: 데이터 트래픽은 `infra_net` 내부로 격리됨.
- **Storage**: 모든 데이터는 `${DEFAULT_DATA_DIR}` 하위의 볼륨에 영구 저장됨.

## Non-functional Requirements

- **Scalability**: 데이터량 증가에 따른 노드 확장이 수동 또는 세미 오토로 가능해야 함.
- **Observability**: Prometheus Exporter를 통해 각 DB의 메트릭을 노출함.
- **Safety**: HAProxy(pg-router)를 통한 단일 진입점(Read/Write Separation) 제공.
