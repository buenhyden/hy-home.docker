# PRD - 11-laboratory (Management & Laboratory)

## Vision

`11-laboratory` 계층은 시스템 관리자 및 개발자를 위한 통합 제어 센터와 실험적인 고립 환경을 제공한다. 분산된 인프라 서비스들을 시각화하고, 컨테이너 및 데이터 리소스에 대한 직관적인 관리 인터페이스를 구축하여 운영 효율성을 극대화한다.

## Personas

- **System Administrator**: 전체 컨테이너 상태를 모니터링하고 가동 중인 서비스들을 제어한다.
- **Backend Developer**: Redis 등 데이터 저장소의 데이터를 시각적으로 확인하고 디버깅한다.
- **AI Agent**: 인프라 구성 정보를 파악하고 서비스 접근점을 확인한다.

## Key Use Cases

- **통합 대시보드 접근**: Homer를 통해 모든 인프라 서비스에 원클릭으로 접근한다.
- **컨테이너 관리**: Portainer GUI를 사용하여 터미널 없이 컨테이너 로그 확인 및 재시작을 수행한다.
- **데이터 시각화**: RedisInsight를 통해 Redis 클러스터의 키 분 분포 및 성능을 분석한다.

## Functional Requirements

- **Service Discovery Display**: 모든 활성 인프라 서비스는 대시보드에 자동으로 또는 수동 설정을 통해 노출되어야 한다.
- **Role-Based Access Control**: 모든 관리 도구는 Keycloak SSO와 연동되어 인증된 사용자만 접근할 수 있어야 한다.
- **Resource Monitoring Integration**: Portainer를 통해 각 서비스의 CPU/Memory 사용량을 실시간으로 확인 가능해야 한다.

## Success Criteria

- 모든 `infra/*` 서비스가 대시보드(Homer)에 누락 없이 등록됨.
- Portainer와 RedisInsight가 Traefik sso-auth 미들웨어를 통해 정상적으로 보호됨.
- 운영자의 인프라 상태 확인 시간이 명령줄 방식 대비 50% 이상 단축됨.

## Scope and Non-goals

- **In-Scope**: Portainer, RedisInsight, Homer Dashboard 구성 및 연동.
- **Non-Goals**: 개별 비즈니스 애플리케이션의 관리 UI(각 앱 레이어에서 담당), 하드웨어 수준의 모니터링(06-observability 담당).
