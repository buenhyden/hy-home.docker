# Qdrant

> High-performance vector similarity search engine.

## 1. Context & Objective (SSoT)

The `qdrant` service provides the vector database layer for AI/ML applications, enabling fast semantic search for high-dimensional embeddings. It is a critical component for Retrieval-Augmented Generation (RAG) workflows.

- **Status**: Production / AI
- **Role**: RAG Persistence
- **SSoT Documentation**: [03.specialized-dbs.md](../../../docs/07.guides/04-data/03.specialized-dbs.md)

## 2. Requirements & Constraints

- **Dependency**: High-dimensional vector embeddings must be generated by upstream models.
- **Port**:
  - `6333`: HTTP REST API
  - `6334`: gRPC High-performance API
- **Security**: API access secured via `QDRANT_API_KEY`.

## 3. Setup & Installation

The service is part of the `ai` and `data` profiles.

```bash
docker compose --profile ai up -d qdrant
```

### Persistence

- **Volume**: `qdrant-data` (`/qdrant/storage`)
- **Host Path**: `${DEFAULT_DATA_DIR}/qdrant/data`

## 4. Usage & Integration

### Integration Points

- **REST API**: `https://qdrant.${DEFAULT_URL}`
- **gRPC**: `qdrant-grpc.${DEFAULT_URL}:${QDRANT_GRPC_PORT:-6334}`

### Snapshots

- **Path**: Automated snapshots are stored in `/qdrant/storage/snapshots`.
- **Temp**: Snapshot generation uses `/tmp` (tmpfs).

## 5. Maintenance & Safety

- **Backups**: Standard volume backups and Qdrant-native snapshot API.
- **Optimization**: Monitor collection indexing status and segment optimization.
- **Health**: Monitor via `/readyz` endpoint.
